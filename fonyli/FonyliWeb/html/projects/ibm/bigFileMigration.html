Big File Migration
Tony Li
Project Introduction
This projects are used for the transition of XXX
on- premises account to YYY Cloud service.
This tool make it easier for customer to easily
and successfully adopt usage of the YYY Cloud
service.
Tools Migration Capability
Current Migration Capability
We currently only support migration for hybrid customers in YYY.
Both full and partial migrations are supported. The current
migration tooling and mechanisms are complex and therefore
always require a trained ZZZ person or business partner to manage
the migration.
The current migration method therefore imposes significant costs
on the customer and this has been identified as a barrier to
adoption of the service by customers.
Expected Migration Capability
The business requirement is that we design a simple, reliable,
repeatable process and tooling which will enable a customer
administrator to migrate data to YYY by themselves.

Architecture
Details
Multi-Thread Safe
1. info.txt is the critical section, we add
ReentrantReadWriteLock to keep it read write safe
and fast
2. Server thread pool can serve different clients
simultaneously
Https transfer speed
1. It is slow to transfer big file via https, so we cut it
into small pieces
2. Check md5 to make sure the merged file is exactly
same as the original file
